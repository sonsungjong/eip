## 📝 통합 아키텍처 (EAI & ESB)

**Q3.** EAI (Enterprise Application Integration) 구축 유형 4가지를 쓰고 각 특징을 연결하시오.

| EAI 유형 | 특징? |
|:---|:---|
| (1) Point-to-Point | ? |
| (2) Hub & Spoke | ? |
| (3) Message Bus | ? |
| (4) Hybrid | ? |

<details>
<summary>정답 보기</summary>

| EAI 유형 | 특징 |
|:---|:---|
| (1) Point-to-Point | 가장 기초적인 1:1 통합 방법 |
| (2) Hub & Spoke | 중앙 집중식 방식, 허브 장애 시 전체에 영향을 미침 |
| (3) Message Bus | 미들웨어(버스)를 두어 연계하는 방식, 확장성과 대용량 처리 우수 |
| (4) Hybrid | 그룹 내는 Hub & Spoke, 그룹 간에는 Message Bus 방식을 사용 |

</details>
