## 📝 증감 연산자 (전위/후위)

**Q1.** 다음 C 프로그램이 실행되었을 때 콘솔 화면에 출력될 결과를 순서대로 쓰시오. (공백 포함)

```c
#include <stdio.h>

int main() {
    int num = 10;
    printf("%d ", ++num);
    printf("%d ", num++);
    printf("%d", num);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**11 11 12**

*(해설: `++num`은 전위 연산자이므로 먼저 1 증가한 후 11이 출력됩니다. `num++`은 후위 연산자이므로 현재 값인 11이 먼저 출력된 후 변수 값이 12로 증가합니다. 마지막으로 `num`을 출력하면 증가된 12가 출력됩니다.)*
</details>

## 📝 비트 연산과 시프트 연산

**Q2.** 다음 코드의 출력 결과를 쓰시오.

```c
#include <stdio.h>

int main() {
    unsigned int a = 15;
    unsigned int b = 22;

    printf("%d ", a & b);
    printf("%d", a << 1);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**6 30**

*(해설: `15`는 이진수로 `01111`, `22`는 `10110` 입니다. 두 값을 AND(`&`) 연산하면 `00110`이 되어 10진수 6이 출력됩니다. `a << 1`은 비트를 왼쪽으로 1칸 단위 이동시키므로 $15 \times 2 = 30$ 이라는 값이 도출됩니다.)*
</details>

```c
#include <stdio.h>

int main() {
    int number = 10;
    int *ptr = &number;

    printf("%d ", *ptr);

```

<details>
<summary>정답 및 해설 보기</summary>

**10 11 11**

*(해설: `*ptr`은 `number`의 값 10을 가리키므로 10 출력, `++*ptr`은 포인터가 가리키는 실제 정수 값을 1 증가시키므로 11 출력, 이후 원본 변수 `number` 역시 같은 메모리의 값이 변경되었으므로 동일하게 11이 출력됩니다.)*
</details>

## 📝 배열과 1차원 포인터 연산

**Q4.** 다음 C 코드가 실행될 때, 화면에 출력되는 숫자들을 순서대로 적으시오.

```c
#include <stdio.h>

int main() {
    int arr1d[5] = {10, 20, 30, 40, 50};

    printf("%d ", *(p + 1));
    printf("%d", *(arr1d + 3));
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**20 40**

*(해설: 배열명 `arr1d`와 포인터 `p`는 동일하게 배열의 시작 주소를 나타냅니다. `*(p + 1)`은 인덱스 1인 `arr1d[1]`과 같아 20을 출력하고, `*(arr1d + 3)`은 `arr1d[3]`과 같아 40을 출력합니다.)*
</details>

## 📝 2차원 배열과 포인터 접근

**Q5.** 다음 2차원 배열 코드 실행 시 콘솔에 출력될 숫자는 무엇인가?

```c
#include <stdio.h>

int main() {
    int arr2d[2][3] = { {1, 2, 3}, {4, 5, 6} };
    int (*pp)[3] = arr2d;

    printf("%d", *(*(pp + 1) + 2));
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**6**

*(해설: 포인터 기반으로 2차원 배열을 접근하는 식입니다. `*(*(pp + 1) + 2)`는 `arr2d[1][2]`와 동일한 형태입니다. 따라서 1행 2열 요소인 6이 출력됩니다.)*
</details>

## 📝 포인터를 이용한 값 교환 (Call by reference)

**Q6.** 다음 코드가 계산하여 최종적으로 출력하는 결과 문자열을 적어보시오.

```c
#include <stdio.h>

void func(int x, int *y) {
    ++x;
    ++*y;
    printf("func_x=%d, func_y=%d\n", x, *y);
}

int main() {
    int i = 1, j = 1;
    func(i, &j);
    printf("main_i=%d, main_j=%d", i, j);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**func_x=2, func_y=2 <br> main_i=1, main_j=2**

*(해설: `func` 함수 매개변수 중 `x`는 값 호출(Call by value)되어 단순히 전역 `i`의 값을 복사받으므로 함수 종료 후 `i`는 본래 값 1이 유지됩니다. 반면 `y`는 주소를 직접 전달받는 참조 호출(Call by reference) 형태로 원본 `j`의 주소 값을 받아 `++*y` 연산이 적용되면서 전역의 `j` 값도 2로 변경됩니다.)*
</details>

## 📝 재귀함수 (호출 스택)

**Q7.** 다음 팩토리얼(Factorial)을 구하는 재귀 호출 함수 코드가 실행될 때의 결과를 쓰시오.

```c
#include <stdio.h>

int calc(int n) {
    if(n == 0 || n == 1) { 
        return 1; 
    } else {
        return n * calc(n - 1);
    }
}

int main() {
    int num = 4;
    int result = calc(num);
    printf("결과는 %d", result);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**결과는 24**

*(해설: 재귀함수가 순환하며 `$4 \times calc(3)$`, `$3 \times calc(2)$`, `$2 \times calc(1)$`로 누적되어 계산됩니다. 최종적으로 `$4 \times 3 \times 2 \times 1 = 24$` 연산이 수행됩니다.)*
</details>

## 📝 구조체

**Q8.** 다음 코드는 연결 리스트(Linked List) 구조에서 인접한 두 노드의 값을 교환하는 로직을 보여준다. 최종 출력 내용을 쓰시오. (단, 변수 값 교환 과정을 눈으로 따라가 볼 것)

```c
#include <stdio.h>

struct Node {
    int value;
    struct Node* next;
};

void swapValues(struct Node* node) {
    while (node != NULL && node->next != NULL) {
        int temp = node->value;
        node->value = node->next->value;
        node->next->value = temp;
        node = node->next->next;
    }
}

int main() {
    struct Node a = {10, NULL};
    struct Node b = {20, NULL};
    struct Node c = {30, NULL};
    
    a.next = &b;
    b.next = &c;

    swapValues(&a);

    struct Node* ptr = &a;
    while (ptr != NULL) {
        printf("%d ", ptr->value);
        ptr = ptr->next;
    }
    
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**20 10 30 ** *(뒤에 공백 하나 포함)*

*(해설: 노드 들은 `a -> b -> c` 순서로 연결되며 초기값은 `10, 20, 30`입니다. `swapValues` 함수가 실행되면 `while` 루프를 통해 쌍(Pair)으로 값을 교환합니다. 첫 번째 루프에서 `a`와 `b`의 값을 교환하여 리스트의 값은 `20, 10, 30`이 됩니다. 이후 `node = node->next->next;`에 의해 반복기준 `node`는 `c`를 가리키게 됩니다. `c`의 `next`가 `NULL`이므로 다음 루프 조건(`node->next != NULL`)을 만족하지 못하여 종료됩니다. 리스트를 순회하며 출력하면 `20 10 30 `이 출력됩니다.)*
</details>

## 📝 서식 지정자를 활용한 출력 (printf)

**Q9.** 다음 코드가 실행되었을 때 콘솔에 어떻게 출력되는지 공백을 포함하여 정확히 적으시오. (단축된 줄바꿈 없이 한 줄로 출력 형태만 작성할 것)

```c
#include <stdio.h>

int main() {
    int num = 10;
    float pi = 3.14159;

    printf("[%5d] ", num);
    printf("[%-5d] ", num);
    printf("%.2f ", pi);
    printf("%X", 255);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**[   10] [10   ] 3.14 FF**

*(해설: `%5d`는 5칸을 확보하고 오른쪽 정렬로 `10`을 출력하므로 자리수 앞 3칸이 공백입니다. `%-5d`는 5칸을 확보하고 왼쪽 정렬이므로 `10` 뒤에 3칸의 공백이 생깁니다. `%.2f`는 소수점 아래 둘째 자리까지 반올림하여 출력하므로 `3.14`가 됩니다. `%X`는 255를 대문자 16진수로 출력하므로 `FF`가 됩니다.)*
</details>

## 📝 변수 스코프 (static 변수)

**Q10.** 다음 코드의 최종 출력 결과를 순서대로 적으시오.

```c
#include <stdio.h>

void counter() {
    static int s_num = 0;
    int a_num = 0;
    s_num++;
    a_num++;
    printf("%d %d | ", s_num, a_num);
}

int main() {
    counter();
    counter();
    counter();
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**1 1 | 2 1 | 3 1 | **

*(해설: `s_num`은 `static` 키워드로 선언되어 메모리 공간을 프로그램 종료 시까지 계속 유지하며, 함수가 호출될 때마다 이전 값이 보존되어 누적(1, 2, 3)됩니다. 반면 `a_num`은 일반 지역 변수로 함수가 호출될 때마다 매번 0으로 새로 초기화된 후 1이 되므로 항상 1이 출력됩니다.)*
</details>

## 📝 switch-case 문과 break (Fall-through)

**Q11.** 다음 코드의 최종 출력 결과를 적으시오.

```c
#include <stdio.h>

int main() {
    int number = 2;
    int sum = 0;

    switch(number) {
        case 1:
            sum += 1;
        case 2:
            sum += 2;
        case 3:
            sum += 3;
            break;
        default:
            sum += 10;
    }
    
    printf("%d", sum);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**5**

*(해설: `number`가 2이므로 `case 2:`로 진입합니다. `sum`은 0에서 2가 더해져 2가 됩니다. 하지만 `case 2` 다음에 `break` 문이 없어서 곧바로 다음 구문인 `case 3:`도 연이어 실행됩니다(Fall-through 현상). `sum`에 3이 추가로 더해져 5가 되며, 그 후 `break`를 만나 switch 문을 완전히 빠져나옵니다.)*
</details>

## 📝 반복문 제어 흐름 (break, continue)

**Q12.** 다음 루프 기반 코드가 실행되었을 때, 콘솔에 출력되는 `count` 변수 값들을 차례대로 적으시오.

```c
#include <stdio.h>

int main() {
    int count = 0;
    while(count < 10) {
        count++;

        if(count % 2 == 0) {
            continue;
        }

        if(count > 5) {
            break;
        }

        printf("%d ", count);
    }
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**1 3 5 **

*(해설: `count`가 홀수일 때는 맨 아래 `printf`가 정상적으로 실행되고, 짝수일 때는 `continue`를 만나 아래 코드를 건너뛰고 바로 다음 루프로 넘어갑니다. 즉 1, 3, 5가 출력됩니다. `count`가 6이 되면 `continue`에 의해 건너뛰게 되며, 7이 되면 `count > 5` 조건을 만족하여 `break`에 의해 루프가 완전히 종료됩니다.)*
</details>

## 📝 공용체 (Union) 메모리 덮어쓰기

**Q13.** 다음은 메모리를 공유하는 공용체에 관한 코드이다. 출력되는 결과를 예상해보시오. 

```c
#include <stdio.h>

union Data {
    int i;
    char c;
};

int main() {
    union Data data;
    data.i = 65;
    
    // (아스키코드 65는 대문자 'A'와 같다)
    printf("%d ", data.i);
    printf("%c", data.c);
    
    data.c = 'B'; // 'B'의 아스키코드는 66
    
    printf(" | %d", data.i);
    return 0;
}
```

<details>
<summary>정답 및 해설 보기</summary>

**65 A | 66**

*(해설: 공용체 `union`은 내부 멤버들(`int i`, `char c`)이 가장 큰 자료형의 같은 메모리 공간을 공유합니다. `data.i`에 정수 65를 넣었을 때 같은 공간을 문자로 읽어오는 `data.c`는 아스키코드 65에 해당하는 'A'가 출력됩니다. 이후 `data.c`에 문자 'B'를 덮어 씌우면, 메모리 값이 변경되어 다시 `data.i`를 읽었을 때 아스키코드 'B'와 매칭되는 정수 66이 출력됩니다.)*
</details>
