# [6과목] 화면 설계

---

## 1. UI (User Interface)와 UX (User Experience)
* **UI (사용자 인터페이스):** 사람과 시스템(컴퓨터, 모바일 등) 간에 의사소통을 할 수 있도록 만들어진 물리적, 가상적 매개체. (버튼, 레이아웃, 화면 구조 등)
* **UX (사용자 경험):** 사용자가 시스템이나 서비스를 이용하면서 느끼고 생각하는 총체적인 경험. UI를 포함하는 더 넓은 개념.

### UI 유형 
| 유형 | 설명 | 예시 |
| :--- | :--- | :--- |
| **CLI (Command Line Interface)** | 명령어를 텍스트로 직접 입력하여 조작 | DOS, 리눅스 터미널 |
| **GUI (Graphical User Interface)** | 그래픽(아이콘, 메뉴 등)을 사용하여 마우스로 조작 | Windows, macOS |
| **NUI (Natural User Interface)** | 사용자의 말, 몸짓 등 **자연스러운 행동**으로 조작 | 음성 인식, 제스처 인식 |
| **OUI (Organic User Interface)** | 입력장치가 곡면이나 유연한 디스플레이 등 **유기적 형태**를 가짐 | 플렉서블 디스플레이, 웨어러블 |

---

## 2. UI 설계 원칙 

| 원칙 | 설명 |
| :--- | :--- |
| **직관성 (Intuitiveness)** | 누구나 쉽게 이해하고 사용할 수 있어야 한다 |
| **유효성 (Effectiveness)** | 사용자의 목적을 정확하고 완벽하게 달성할 수 있어야 한다 |
| **학습성 (Learnability)** | 누구나 쉽게 배우고 익힐 수 있어야 한다 |
| **유연성 (Flexibility)** | 사용자의 요구사항을 최대한 수용하고, 오류를 최소화해야 한다 |

---

## 3. 화면 설계 산출물 (UI 설계 도구) 

| 종류 | 설명 및 특징 |
| :--- | :--- |
| **와이어프레임 (Wireframe)** | 기획 단계 초기에 제작하는 **화면 단위의 뼈대(레이아웃) 스케치**. 선과 도형으로 화면의 구조만 단순하게 표현 |
| **목업 (Mockup)** | 와이어프레임보다 실제 화면과 유사하게 만든 **정적인 형태의 모형**. 시각적 디자인이 적용되지만 동적 기능은 없음 |
| **스토리보드 (Storyboard)** | 와이어프레임에 콘텐츠 설명, 페이지 간 이동 흐름 등을 추가한 문서. **디자이너와 개발자가 최종 참고하는 작업 지침서** |
| **프로토타입 (Prototype)** | 인터랙션을 적용하여 실제 구현된 것처럼 **테스트가 가능한 동적인 형태의 모형**. 사용성 테스트나 요구사항 검증용 |
| **유스케이스 (Use Case)** | 사용자 측면에서의 요구사항으로, 사용자가 목표를 달성하기 위해 수행할 내용을 기술한 것 |

> **산출물 발전 순서:** 와이어프레임 → 목업 → 스토리보드 → 프로토타입

## 4. UI 설계 프로세스
1. **문제 정의:** 시스템의 목적과 사용자의 요구사항 파악
2. **사용자 모델 조사:** 타겟 사용자의 특성, 환경, 숙련도 등 분석
3. **작업 분석:** 사용자가 시스템을 통해 수행할 작업의 순서와 절차 분석
4. **컴퓨터 오브젝트 및 기능 정의:** 작업에 필요한 화면 요소와 기능 정의
5. **사용자 인터페이스 정의:** 실제 화면 디자인 및 레이아웃 설계
6. **디자인 평가:** 설계된 UI의 사용성 및 문제점 평가

## 5. 품질 요구사항 (ISO/IEC 9126) 
소프트웨어의 품질 특성을 평가하기 위한 국제 표준입니다.

| 품질 특성 | 설명 |
| :--- | :--- |
| **기능성 (Functionality)** | 요구사항을 정확하게 만족하는 기능을 제공하는 능력 |
| **신뢰성 (Reliability)** | 규정된 조건에서 주어진 기간 동안 오류 없이 기능을 수행하는 능력 |
| **사용성 (Usability)** | 사용자가 쉽게 이해하고, 배우고, 사용할 수 있는 능력 |
| **효율성 (Efficiency)** | 할당된 시간과 자원 내에서 적절한 성능을 제공하는 능력 |
| **유지보수성 (Maintainability)** | 요구사항 변경, 오류 수정 등 환경 변화에 소프트웨어를 쉽게 수정할 수 있는 능력 |
| **이식성 (Portability)** | 다양한 하드웨어/소프트웨어 환경으로 쉽게 이전할 수 있는 능력 |

> 암기: **기·신·사·효·유·이**

## 6. UI 테스트 기법

| 기법 | 설명 |
| :--- | :--- |
| **사용성 테스트 (Usability Test)** | 실제 사용자가 직접 제품을 사용하면서 문제점을 찾는 테스트 |
| **휴리스틱 평가 (Heuristic Evaluation)** | 전문가가 경험적 원칙(휴리스틱)에 기반하여 UI의 문제점을 평가 |
| **A/B 테스트** | 두 가지 이상의 디자인 안(A안, B안)을 실제 사용자에게 노출하여 성과를 비교 |

## 7. 웹/모바일 화면 설계 시 고려사항

| 항목 | 설명 |
| :--- | :--- |
| **반응형 웹 (Responsive Web)** | 디스플레이 종류에 따라 화면 크기가 자동으로 최적화되는 웹 구현 기법 |
| **크로스 브라우징 (Cross Browsing)** | 서로 다른 브라우저에서도 동일한 화면과 기능을 제공하는 웹 표준 기술 |
| **웹 접근성 (Web Accessibility)** | 장애인이나 고령자 등도 웹 정보에 동등하게 접근하고 이해할 수 있도록 보장하는 것 |

### 웹 접근성 지침 (WCAG) 4가지 원칙 
| 원칙 | 설명 |
| :--- | :--- |
| **인식의 용이성 (Perceivable)** | 모든 콘텐츠는 사용자가 인식할 수 있어야 함 (대체 텍스트, 자막 등) |
| **운용의 용이성 (Operable)** | 사용자 인터페이스 구성요소는 조작 가능하고 내비게이션할 수 있어야 함 |
| **이해의 용이성 (Understandable)** | 정보와 사용자 인터페이스 조작은 이해할 수 있어야 함 |
| **견고성 (Robust)** | 다양한 사용자 에이전트(보조 기술 포함)에서 해석될 수 있어야 함 |

## 8. HCI (Human Computer Interaction)
사람과 컴퓨터 사이의 상호작용을 연구하여, 사람들이 컴퓨터 시스템을 더 쉽고 편리하게 사용할 수 있도록 시스템을 디자인하고 평가하는 학문입니다.

## 9. 감성공학 (Kansei Engineering)
인간의 감성(느낌, 이미지 등)을 정량적으로 측정하고 분석하여, 이를 제품이나 환경 설계에 반영하는 기술입니다. 인간의 삶의 질을 향상시키는 것이 목적입니다.

## ★ 핵심 암기 포인트 ★

1. **UI 유형:** CLI(명령어) → GUI(그래픽) → NUI(자연스러운 행동) → OUI(유기적 형태)
2. **UI 설계 원칙:** 직관성, 유효성, 학습성, 유연성
3. **산출물 순서:** 와이어프레임(뼈대) → 목업(정적 모형) → 스토리보드(작업 지침서) → 프로토타입(동적 모형)
4. **ISO/IEC 9126 품질 특성:** 기능성, 신뢰성, 사용성, 효율성, 유지보수성, 이식성
5. **웹 접근성(WCAG) 4원칙:** 인식의 용이성, 운용의 용이성, 이해의 용이성, 견고성
6. **스토리보드** = 디자이너와 개발자가 최종 참고하는 작업 지침서
