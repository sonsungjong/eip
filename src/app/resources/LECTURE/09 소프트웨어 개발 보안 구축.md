# [9과목] 소프트웨어 개발 보안 구축

---

## 1. 정보 보안의 3대 요소 (CIA Triad)
*(암기: 기·무·가)*

| 요소 | 설명 | 보안 기술 예시 |
| :--- | :--- | :--- |
| **기밀성 (Confidentiality)** | 인가된 사용자만 정보 자산에 접근할 수 있음 | 방화벽, 암호화 |
| **무결성 (Integrity)** | 인가된 사용자만 정보를 수정할 수 있으며, 전송 중 변조되지 않았음을 보장 | 해시 함수, 디지털 서명 |
| **가용성 (Availability)** | 인가된 사용자는 언제든지 정보에 접근하여 사용할 수 있음 | 데이터 백업, 이중화 |

---

## 2. Secure SDLC (안전한 소프트웨어 개발 생명주기)
SDLC에 보안 강화를 위한 프로세스를 포함한 것입니다.

| 방법론 | 핵심 키워드 | 설명 |
| :--- | :--- | :--- |
| **Seven Touchpoints** | **개발자 중심**, 7가지 보안 활동 | SDLC 각 단계에 7가지 보안 강화 활동을 개발자에게 집중 관리하도록 요구. 소프트웨어 보안의 모범사례를 SDLC에 통합 |
| **MS SDL** | **마이크로소프트**, 자체 방법론 | 마이크로소프트사에서 자체 수립한 안전한 소프트웨어 개발 방법론. Windows 등 자사 제품에 적용 |
| **CLASP** | **활동 중심, 역할 기반** | 프로젝트 관리자, 설계자, 개발자 등 각 역할별로 보안 활동을 정의. 현재 운영 중인 시스템에 적용하기 쉬움 |

> **구분 포인트:** Seven Touchpoints = "개발자"가 보안을 책임, CLASP = "역할별"로 보안 활동을 분담

## 3. 주요 네트워크 공격 기법

### DoS (Denial of Service) 공격
시스템 자원을 부족하게 하여 정상적인 서비스를 방해하는 공격입니다.

| 공격 기법 | 설명 |
| :--- | :--- |
| **Ping of Death** | 정상 크기보다 아주 **큰 ICMP 패킷**을 보내어 네트워크를 마비 |
| **Smurfing (스머핑)** | IP/ICMP 특성을 악용하여 **브로드캐스트 주소**로 대량 데이터를 보내 네트워크 마비 |
| **SYN Flooding** | TCP **3-Way Handshake** 취약점을 이용. 대량의 SYN 패킷으로 서버의 대기 큐를 가득 채움 |
| **TearDrop** | IP 패킷의 **Fragment Offset 값을 조작**하여 재조립 시 오류·과부하 유발 |
| **Land Attack** | 출발지 IP와 목적지 IP를 **동일하게** 설정하여 자기 자신에게 응답하게 하여 무한 루프 유발 |

### DDoS (Distributed DoS) 공격
여러 대의 **분산된 공격자(좀비PC)**를 이용하여 동시에 DoS 공격을 수행하는 형태입니다.

| 용어 | 설명 |
| :--- | :--- |
| **봇넷 (Botnet)** | 악성 코드에 감염되어 원격 조종이 가능한 좀비PC들의 네트워크 |
| **좀비PC** | 봇넷에 포함되어 공격자의 명령을 수행하는 감염된 컴퓨터 |
| **C&C 서버** | 봇넷을 제어하는 명령·제어(Command & Control) 서버 |

## 4. 주요 웹 애플리케이션 공격 기법

| 공격 기법 | 설명 |
| :--- | :--- |
| **SQL Injection** | 입력 폼에 **악의적인 SQL 구문**을 삽입하여 DB를 비정상적으로 조작 |
| **XSS (Cross-Site Scripting)** | 게시판 등에 **악성 스크립트**를 삽입하여 사용자 정보(쿠키, 세션 등) 탈취 |
| **CSRF (Cross-Site Request Forgery)** | 사용자의 의지와 무관하게 **공격자가 의도한 위조 요청**을 보내게 함 |

## 5. 기타 보안 공격 및 악성 프로그램

### 시스템/네트워크 공격 기법

| 공격 기법 | 설명 |
| :--- | :--- |
| **스니핑 (Sniffing)** | 네트워크 상에서 전송되는 **패킷을 몰래 엿보는** 수동적 공격 |
| **스푸핑 (Spoofing)** | IP 주소, MAC 주소, 이메일 등 **출처를 위조**하여 속이는 공격 (IP Spoofing, ARP Spoofing, DNS Spoofing 등) |
| **세션 하이재킹 (Session Hijacking)** | 인증이 완료된 사용자의 **세션을 가로채** 정상 사용자로 위장하는 공격 |
| **버퍼 오버플로우 (Buffer Overflow)** | 메모리 버퍼의 크기를 초과하는 데이터를 입력하여 **인접 메모리를 덮어쓰는** 공격 |
| **포맷 스트링 공격 (Format String)** | printf() 등의 함수에서 **포맷 스트링을 악용**하여 메모리를 읽거나 쓰는 공격 |
| **레이스 컨디션 (Race Condition)** | 다수의 프로세스가 동시에 같은 자원에 접근할 때 **실행 순서에 따라 비정상적 결과**가 발생하는 취약점 |
| **백도어 (Backdoor)** | 시스템에 **비인가된 접근 경로**를 만들어 두는 것 |
| **사회공학 (Social Engineering)** | 사람의 심리적 취약점을 이용하여 비밀 정보를 취득하는 **비기술적 공격** |

### 악성 프로그램 (Malware)

| 종류 | 설명 |
| :--- | :--- |
| **웜 (Worm)** | 스스로 복제하여 네트워크를 통해 전파되는 악성 프로그램. **자기 복제**, 숙주 불필요 |
| **트로이목마 (Trojan Horse)** | 정상 프로그램으로 위장하여 설치된 후 악의적 기능을 수행. **자기 복제 안 함** |
| **랜섬웨어 (Ransomware)** | 파일을 암호화한 뒤 해독 키를 빌미로 **금전을 요구**하는 악성 프로그램 |
| **키로거 (Key Logger)** | 사용자의 **키보드 입력을 몰래 기록**하여 비밀번호 등을 탈취 |
| **루트킷 (Rootkit)** | 시스템에 침입한 후 자신의 존재를 **은폐하는 도구 모음** |
| **애드웨어 (Adware)** | 사용자 동의 없이 **광고를 강제로 표시**하는 프로그램 |
| **스파이웨어 (Spyware)** | 사용자의 동의 없이 **개인 정보를 수집**하여 외부로 전송하는 프로그램 |

### 제로 데이 공격 (Zero Day Attack)
보안 취약점이 발견되었을 때 **패치가 배포되기 전에** 해당 취약점을 이용하여 이루어지는 공격.

### 기타 주요 공격 용어

| 용어 | 설명 |
| :--- | :--- |
| **피싱 (Phishing)** | 금융기관 등을 사칭한 이메일/사이트로 유도하여 개인 정보를 탈취 |
| **스피어 피싱 (Spear Phishing)** | 특정 개인/조직을 대상으로 한 **맞춤형** 피싱 공격 |
| **스미싱 (Smishing)** | **SMS + Phishing**. 문자 메시지를 이용한 피싱 |
| **파밍 (Pharming)** | DNS를 변조하여 **정상 사이트로 접속해도 가짜 사이트**로 연결되게 하는 공격 |
| **큐싱 (Qshing)** | **QR코드 + Phishing**. 악성 QR코드를 이용한 피싱 |
| **APT (Advanced Persistent Threat)** | 특정 타깃을 대상으로 **장기간에 걸쳐 지속적**으로 다양한 공격을 수행하는 지능형 위협 |
| **워터링 홀 (Watering Hole)** | 타깃이 자주 방문하는 **웹사이트를 감염**시켜 악성 코드를 유포 |
| **드라이브 바이 다운로드 (Drive-by Download)** | 웹 페이지 방문만으로 **사용자 모르게 악성 코드가 자동 다운로드** |

## 6. 서버 접근 통제 (Access Control)

| 기법 | 기준 | 특징 |
| :--- | :--- | :--- |
| **DAC (임의 접근 통제)** | 사용자의 **신원(Identity)** | 데이터 **소유자**가 권한을 지정 |
| **MAC (강제 접근 통제)** | **보안 등급(Label)** | **시스템/관리자**가 권한을 지정. 군사/정부 시스템 |
| **RBAC (역할 기반 접근 통제)** | 사용자의 **역할(Role)** | 인사 이동이 잦은 조직에 적합. 가장 널리 사용 |

## 7. 인증 기술

### 인증 유형

| 유형 | 설명 | 예시 |
| :--- | :--- | :--- |
| **지식 기반 (Something You Know)** | 사용자가 알고 있는 것 | 비밀번호, PIN |
| **소유 기반 (Something You Have)** | 사용자가 가지고 있는 것 | OTP 토큰, 스마트카드, 휴대폰 |
| **생체 기반 (Something You Are)** | 사용자의 생체 정보 | 지문, 홍채, 얼굴 인식 |
| **행위 기반 (Something You Do)** | 사용자의 행동 패턴 | 서명, 타이핑 패턴 |

> **다중 인증(MFA):** 2가지 이상의 인증 유형을 결합하여 보안을 강화하는 방식

### 주요 인증 기술

| 기술 | 설명 |
| :--- | :--- |
| **SSO (Single Sign-On)** | 한 번의 인증으로 여러 시스템/서비스에 접근할 수 있는 기술 |
| **OAuth** | 제3자 애플리케이션이 사용자의 자원에 접근할 수 있도록 권한을 위임하는 개방형 표준 프로토콜 |
| **커버로스 (Kerberos)** | 티켓(Ticket) 기반의 네트워크 인증 프로토콜. 대칭키 암호화 사용 |

## 8. 암호화 관련 보안 기술

| 기술 | 설명 |
| :--- | :--- |
| **솔트 (Salt)** | 해시 함수에 원본 데이터와 함께 추가하는 **무작위 데이터**. 동일 비밀번호도 다른 해시값 생성 |
| **키 스트레칭 (Key Stretching)** | 해시 함수를 **여러 번 반복** 적용하여 무차별 대입 공격(Brute Force)을 어렵게 함 |
| **디지털 서명 (Digital Signature)** | 송신자의 **개인키로 서명**하고, 수신자가 **공개키로 검증**하여 무결성·인증·부인 방지를 보장 |
| **PKI (Public Key Infrastructure)** | 인증서의 발급·관리를 수행하는 **공개키 기반구조**. CA(인증기관), RA(등록기관) 등으로 구성 |

## 9. 보안 솔루션

| 솔루션 | 설명 |
| :--- | :--- |
| **방화벽 (Firewall)** | 내·외부 네트워크 간 전송되는 패킷을 **정책에 따라 차단/허용**하는 보안 시스템 |
| **IDS (Intrusion Detection System)** | 네트워크/시스템의 비정상적 사용 및 오용을 **탐지(감시)**하는 시스템 |
| **IPS (Intrusion Prevention System)** | 비정상적 트래픽을 탐지하고 **자동으로 차단**까지 수행하는 시스템 (IDS + 차단) |
| **WAF (Web Application Firewall)** | **웹 애플리케이션** 보안에 특화된 방화벽. SQL Injection, XSS 등을 차단 |
| **VPN (Virtual Private Network)** | 공중 네트워크를 통해 **암호화된 가상의 사설 통신망**을 구축하는 기술 |
| **NAC (Network Access Control)** | 네트워크에 접근하는 **단말기의 보안 상태를 검사**하여 접근을 제어 |
| **DLP (Data Loss Prevention)** | 기업 내부의 **중요 정보가 외부로 유출되는 것을 방지**하는 솔루션 |
| **ESM (Enterprise Security Management)** | 방화벽, IDS 등 **여러 보안 장비를 통합 관리**하는 보안 관제 시스템 |
| **SIEM (Security Information and Event Management)** | 보안 로그와 이벤트를 **실시간 수집·분석**하여 위협을 탐지하는 시스템 |
| **DRM (Digital Rights Management)** | 디지털 콘텐츠의 **저작권을 보호**하기 위한 관리 기술 |

> **IDS vs IPS:** IDS는 탐지만, IPS는 탐지 + 차단

## ★ 핵심 암기 포인트 ★

1. **CIA 3요소:** 기밀성(접근 통제), 무결성(변조 방지), 가용성(서비스 보장)
2. **접근 통제:** DAC(신원/소유자), MAC(등급/관리자), RBAC(역할)
3. **DoS 공격:** Ping of Death(큰 패킷), Smurfing(브로드캐스트), SYN Flooding(3-Way Handshake), TearDrop(Fragment Offset), Land Attack(출발지=목적지)
4. **웹 공격 3대장:** SQL Injection, XSS, CSRF
5. **악성 프로그램:** 웜(자기복제O), 트로이목마(위장/자기복제X), 랜섬웨어(금전 요구)
6. **인증 유형:** 지식(비밀번호), 소유(OTP), 생체(지문), 행위(서명)
7. **피싱 변형:** 피싱(이메일), 스피어피싱(특정 대상), 스미싱(SMS), 파밍(DNS), 큐싱(QR)
8. **보안 솔루션:** 방화벽(차단), IDS(탐지), IPS(탐지+차단), WAF(웹 전용), VPN(암호화 터널), DLP(유출 방지)
9. **Secure SDLC:** Seven Touchpoints(개발자 중심), MS SDL(마이크로소프트), CLASP(활동 중심/역할 기반)
10. **디지털 서명:** 개인키로 서명, 공개키로 검증 → 무결성·인증·부인 방지
