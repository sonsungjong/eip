# [2과목] 데이터 입출력 구현

## 1. 데이터 모델링 (Data Modeling)
현실 세계의 정보들을 컴퓨터에 표현하기 위해서 단순화, 추상화하여 체계적으로 표현한 개념적 모형입니다.

* **개념적 데이터 모델링:** 현실 세계에 대한 인식을 추상적 개념으로 표현. (E-R 다이어그램 작성)
* **논리적 데이터 모델링:** 개념적 구조를 컴퓨터가 이해하고 처리할 수 있는 환경에 맞도록 변환. (관계형 데이터 모델 등)
* **물리적 데이터 모델링:** 실제 데이터베이스에 어떻게 저장할 것인지 물리적 구조를 설계.

### 데이터베이스 설계 절차 ★2026년 1회차★
데이터베이스 설계는 요구사항을 분석한 뒤 개념적 구조, 논리적 구조, 물리적 구조 순서로 구체화하고 구현합니다.

| 순서 | 단계 | 설명 |
| :---: | :--- | :--- |
| 1 | **요구사항 분석** | 사용자 요구사항을 수집하고 데이터베이스에 저장할 대상과 처리 조건을 분석 |
| 2 | **개념적 설계** | 현실 세계의 데이터를 개체, 속성, 관계로 표현. E-R 다이어그램 작성 |
| 3 | **논리적 설계** | 개념적 모델을 DBMS가 처리할 수 있는 논리적 데이터 모델로 변환. 테이블, 속성, 키 설계 |
| 4 | **물리적 설계** | 실제 저장 구조, 인덱스, 접근 경로, 저장 레코드 형식 등을 설계 |
| 5 | **구현** | SQL 등을 사용하여 실제 데이터베이스 객체를 생성 |

> **암기 순서:** 요구사항 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계 → 구현

### E-R 다이어그램 (Entity-Relationship Diagram) ★
개념적 모델링 단계에서 현실 세계를 그림으로 표현하는 도구.

| 구성 요소 | 표기 | 설명 |
| :--- | :--- | :--- |
| **개체 (Entity)** | 사각형 | 독립적으로 존재하는 사물 또는 개념 (예: 학생, 과목) |
| **속성 (Attribute)** | 타원형 | 개체가 가지는 특성 (예: 학번, 이름) |
| **관계 (Relationship)** | 마름모 | 개체 간의 연관성 (예: 수강) |

### 카디널리티 (Cardinality) — 관계의 대응 수
| 종류 | 설명 | 예시 |
| :--- | :--- | :--- |
| **1:1** | 한 개체가 다른 개체 한 개와 대응 | 남편 : 아내 |
| **1:N** | 한 개체가 다른 개체 여러 개와 대응 | 부서 : 사원 |
| **M:N** | 여러 개체가 서로 여러 개와 대응 | 학생 : 과목 |

## 2. 이상 현상 (Anomaly)
테이블 내의 데이터들이 불필요하게 중복되어 발생하게 되는 논리적 오류입니다.

* **삽입 이상 (Insertion Anomaly):** 데이터를 삽입할 때 불필요한 데이터까지 함께 삽입해야만 하는 현상.
    * 예) 신입 사원을 등록하려면 배정될 부서가 반드시 있어야 하는 경우.
* **삭제 이상 (Deletion Anomaly):** 한 튜플을 삭제할 때 연쇄 삭제가 발생하여, 유지해야 할 다른 정보까지 삭제되는 현상.
    * 예) 특정 과목을 수강하는 학생이 한 명뿐일 때, 그 학생을 삭제하면 과목 정보도 함께 사라지는 경우.
* **갱신 이상 (Update Anomaly):** 중복된 튜플들 중 일부만 갱신하여 데이터의 모순(불일치)이 발생하는 현상.
    * 예) 같은 부서명이 여러 행에 중복될 때, 일부만 변경하면 데이터가 불일치하는 경우.

## 3. 정규화 (Normalization) ★중요★
이상 현상을 방지하기 위해 중복성을 최소화하고 정보의 일관성을 보장하기 위해 테이블을 분해하는 과정입니다.
*(암기 팁: 도·부·이·결·다·조)*

| 단계 | 조건 | 설명 |
| :--- | :--- | :--- |
| **1NF** (제1정규형) | **도**메인이 원자값 | 모든 속성 값이 원자값(Atomic Value)만 갖도록 분해 |
| **2NF** (제2정규형) | **부**분 함수 종속 제거 | 기본키가 아닌 모든 속성이 기본키에 완전 함수 종속되도록 분해 |
| **3NF** (제3정규형) | **이**행적 함수 종속 제거 | A → B, B → C 일 때 A → C 가 성립하는 관계를 분해 |
| **BCNF** (보이스-코드) | **결**정자가 모두 후보키 | 모든 결정자가 후보키가 되도록 분해 |
| **4NF** (제4정규형) | **다**중값 종속 제거 | 다치 종속(Multi-Valued Dependency) 관계를 분해 |
| **5NF** (제5정규형) | **조**인 종속성 이용 | 조인 종속이 후보키를 통해서만 성립되도록 분해 |

### 역정규화 (반정규화, Denormalization) ★
정규화된 데이터 모델을 **성능 향상**을 목적으로 의도적으로 중복을 허용하여 테이블 구조를 변경하는 기법.

| 기법 | 설명 |
| :--- | :--- |
| **테이블 병합** | 1:1 또는 1:N 관계의 테이블을 하나로 합침 |
| **테이블 분할** | 자주 사용하는 컬럼과 그렇지 않은 컬럼을 분리 (수직 분할) |
| **중복 칼럼 추가** | 조인 없이 조회하기 위해 다른 테이블의 컬럼을 복제해 추가 |
| **중복 테이블 추가** | 집계·통계용 요약 테이블을 별도로 생성 |

> 주의: 역정규화는 조회 성능은 향상되지만 데이터 일관성 유지 비용이 증가함.

## 4. SQL (Structured Query Language) ★★★ (실기 최빈출)
관계형 데이터베이스를 조작하기 위한 표준 언어입니다.

### SQL 명령어 분류

| 분류 | 명칭 | 명령어 | 설명 |
| :--- | :--- | :--- | :--- |
| **DDL** | 데이터 정의어 | `CREATE` | 테이블, 뷰, 인덱스 등 객체 생성 |
| | | `ALTER` | 객체 구조 변경 |
| | | `DROP` | 객체 삭제 |
| | | `TRUNCATE` | 테이블 내 모든 데이터 삭제 (구조 유지) |
| **DML** | 데이터 조작어 | `SELECT` | 데이터 조회 |
| | | `INSERT` | 데이터 삽입 |
| | | `UPDATE` | 데이터 수정 |
| | | `DELETE` | 데이터 삭제 |
| **DCL** | 데이터 제어어 | `GRANT` | 권한 부여 |
| | | `REVOKE` | 권한 취소 |
| **TCL** | 트랜잭션 제어어 | `COMMIT` | 트랜잭션 확정 (영구 반영) |
| | | `ROLLBACK` | 트랜잭션 취소 (이전 상태로 복구) |
| | | `SAVEPOINT` | 롤백 지점 설정 |

### SELECT 문 기본 구조 (작성 순서)
```sql
SELECT   컬럼명
FROM     테이블명
WHERE    조건
GROUP BY 그룹 기준 컬럼
HAVING   그룹 조건
ORDER BY 정렬 기준 컬럼 [ASC|DESC];
```
*(실행 순서: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY)*

### JOIN 종류
| 종류 | 설명 |
| :--- | :--- |
| **INNER JOIN** | 두 테이블의 교집합. 조건에 일치하는 행만 반환 |
| **LEFT OUTER JOIN** | 왼쪽 테이블 전체 + 오른쪽 일치 행 (불일치 시 NULL) |
| **RIGHT OUTER JOIN** | 오른쪽 테이블 전체 + 왼쪽 일치 행 (불일치 시 NULL) |
| **FULL OUTER JOIN** | 양쪽 테이블 전체 (불일치 시 NULL) |
| **CROSS JOIN** | 두 테이블의 모든 행의 조합 (카테시안 곱) |

## 5. 트랜잭션 (Transaction)
데이터베이스의 상태를 변환시키는 하나의 논리적 기능을 수행하기 위한 작업의 단위입니다.

### 트랜잭션의 특징 (ACID)
* **Atomicity (원자성):** 트랜잭션의 연산은 모두 반영되든지 아니면 전혀 반영되지 않아야 함 (All or Nothing).
* **Consistency (일관성):** 트랜잭션이 성공적으로 완료되면 언제나 일관성 있는 데이터베이스 상태로 변환됨.
* **Isolation (독립성/격리성):** 둘 이상의 트랜잭션이 동시에 실행될 때, 서로의 연산이 끼어들 수 없음.
* **Durability (영속성/지속성):** 성공적으로 완료된 트랜잭션의 결과는 시스템 고장 시에도 영구적으로 반영됨.

### 트랜잭션 상태 5종 ★
| 상태 | 설명 |
| :--- | :--- |
| **활동 (Active)** | 트랜잭션이 실행 중인 상태 |
| **부분 완료 (Partially Committed)** | 마지막 연산이 실행된 직후, COMMIT 전 상태 |
| **완료 (Committed)** | COMMIT이 완료되어 데이터베이스에 반영된 상태 |
| **실패 (Failed)** | 오류 발생으로 트랜잭션 실행이 중단된 상태 |
| **철회 (Aborted)** | ROLLBACK되어 트랜잭션 시작 전 상태로 복구된 상태 |

## 6. 병행 제어 (Concurrency Control) ★
여러 트랜잭션이 동시에 실행될 때 데이터베이스의 일관성을 유지하기 위한 기법입니다.

### 병행 제어 미흡 시 발생하는 문제
| 문제 | 설명 |
| :--- | :--- |
| **갱신 분실 (Lost Update)** | 두 트랜잭션이 같은 데이터를 갱신할 때 한 갱신 결과가 사라짐 |
| **비완료 의존성 (Dirty Read)** | 완료되지 않은 트랜잭션의 데이터를 다른 트랜잭션이 읽음 |
| **모순성 (Inconsistency)** | 같은 데이터를 두 트랜잭션이 동시에 조회할 때 서로 다른 값을 읽음 |
| **연쇄 복귀 (Cascading Rollback)** | 한 트랜잭션 롤백이 다른 트랜잭션에 연쇄적으로 영향을 줌 |

### 병행 제어 기법
| 기법 | 설명 |
| :--- | :--- |
| **로킹 (Locking)** | 데이터에 잠금(Lock)을 설정하여 다른 트랜잭션의 접근을 제한. 공유 락(S-Lock)과 배타 락(X-Lock)으로 구분 |
| **타임스탬프 (Timestamp)** | 트랜잭션 시작 시간을 기록하여 순서를 부여하고 그 순서에 따라 실행 |
| **낙관적 검증 (Optimistic Validation)** | 충돌이 없다고 가정하고 실행 후, 완료 전에 충돌 여부를 검사 |
| **다중 버전 (MVCC)** | 데이터의 여러 버전을 유지하여 읽기와 쓰기가 서로 방해하지 않도록 함 |

> 교착 상태 (Deadlock): 두 트랜잭션이 서로 상대방의 락 해제를 기다리며 무한 대기하는 상태.

## 7. 회복 기법 (Recovery) ★
장애 발생 시 데이터베이스를 일관된 상태로 복구하는 기법입니다.

| 기법 | 설명 |
| :--- | :--- |
| **로그 기반 회복** | 모든 변경 내용을 로그에 기록하여 REDO(재실행) / UNDO(취소)로 복구 |
| **체크포인트 (Checkpoint)** | 주기적으로 체크포인트를 설정하여, 장애 시 해당 시점 이후부터만 복구 |
| **그림자 페이지 (Shadow Paging)** | 변경 전 원본 페이지를 별도 보관, 장애 시 원본으로 복구 |

## 8. 물리 데이터베이스 설계 요소

* **인덱스 (Index):** 데이터 레코드를 빠르게 접근하기 위해 `<키 값, 포인터>` 쌍으로 구성되는 데이터 구조.
    * 클러스터드 인덱스: 데이터 자체가 인덱스 순서대로 정렬 저장. 테이블당 1개만 가능.
    * 넌클러스터드 인덱스: 별도의 인덱스 구조 유지. 테이블당 여러 개 가능.
* **뷰 (View):** 하나 이상의 기본 테이블로부터 유도된 가상 테이블. 보안성 향상, 데이터 독립성 제공.
    * 특징: 독립적인 인덱스를 가질 수 없음. 뷰를 통한 UPDATE·INSERT·DELETE는 제한적.
* **파티션 (Partition):** 대용량 테이블이나 인덱스를 관리하기 쉬운 단위로 분할하는 기능.

| 파티션 종류 | 설명 |
| :--- | :--- |
| **범위 분할 (Range)** | 값의 범위를 기준으로 분할 (예: 날짜별) |
| **해시 분할 (Hash)** | 해시 함수로 균등 분산 |
| **목록 분할 (List)** | 특정 값 목록을 기준으로 분할 |
| **조합 분할 (Composite)** | 위 기법을 두 가지 이상 조합 |
