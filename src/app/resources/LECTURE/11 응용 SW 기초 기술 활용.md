## 1. 운영체제 (OS) 의 목적

| 목적 | 설명 |
| :--- | :--- |
| **처리 능력 (Throughput)** | 일정 시간 내에 시스템이 처리하는 일의 양 |
| **반환 시간 (Turnaround Time)** | 작업 의뢰부터 처리 완료까지 걸린 시간 |
| **사용 가능도 (Availability)** | 시스템을 얼마나 빠르게 사용할 수 있는지의 정도 |
| **신뢰도 (Reliability)** | 주어진 문제를 얼마나 정확하게 해결하는지의 정도 |

## 2. 프로세스 상태 전이

```
생성(New) → 준비(Ready) → 실행(Running) → 완료(Terminated)
                  ↑              ↓
                  ← 대기(Waiting) ←
```

| 전이 | 설명 |
| :--- | :--- |
| **디스패치 (Dispatch)** | 준비 → 실행. 준비 큐에서 CPU를 할당받음 |
| **타임아웃 (Timeout)** | 실행 → 준비. 할당 시간(Time Slice) 초과 |
| **블록 (Block)** | 실행 → 대기. I/O 요청 등으로 인해 CPU를 반납 |
| **웨이크업 (Wake-up)** | 대기 → 준비. I/O 완료 등으로 다시 준비 상태로 전이 |

## 3. 프로세스 스케줄링 알고리즘

### 비선점 (Non-Preemptive) 스케줄링
CPU를 할당받으면 다른 프로세스가 강제로 빼앗을 수 없는 방식입니다.

| 알고리즘 | 설명 |
| :--- | :--- |
| **FCFS (First Come First Served)** | 도착 순서대로 CPU 할당 (FIFO). 콘보이 현상 발생 가능 |
| **SJF (Shortest Job First)** | 실행 시간이 가장 짧은 프로세스 먼저. **기아 현상(Starvation)** 발생 가능 |
| **HRN (Highest Response-ratio Next)** | SJF 보완. 대기 시간 고려. **우선순위 = (대기시간 + 서비스시간) / 서비스시간** |
| **우선순위 (Priority)** | 우선순위가 높은 프로세스 먼저 실행 |

### 선점 (Preemptive) 스케줄링
우선순위가 높은 프로세스가 현재 프로세스를 중단시키고 CPU를 빼앗을 수 있는 방식입니다.

| 알고리즘 | 설명 |
| :--- | :--- |
| **Round Robin (RR)** | 시간 할당량(Time Slice)만큼 CPU 사용 후 다음 프로세스로 교체. 시분할 시스템에 적합 |
| **SRT (Shortest Remaining Time)** | 남은 시간이 가장 짧은 프로세스에게 CPU 할당. SJF의 선점형 버전 |
| **다단계 큐 (Multi-level Queue)** | 프로세스를 그룹별로 여러 큐에 배치하여 스케줄링 |
| **다단계 피드백 큐 (Multi-level Feedback Queue)** | 다단계 큐에서 프로세스가 큐 간 이동 가능. 가장 일반적인 CPU 스케줄링 |

### 스케줄링 성능 지표 (계산 대비)
* **반환 시간(Turnaround Time)** = 완료 시간 - 도착 시간
* **대기 시간(Waiting Time)** = 반환 시간 - 서비스(실행) 시간
* **평균 대기 시간** = 각 프로세스의 대기 시간 합 / 프로세스 수

## 4. 교착상태 (Deadlock)
둘 이상의 프로세스가 서로 상대방의 자원을 기다리면서 무한히 대기하는 상태입니다.

### 발생 조건 4가지 (모두 만족해야 교착상태 발생)

| 조건 | 설명 |
| :--- | :--- |
| **상호 배제 (Mutual Exclusion)** | 한 자원은 한 번에 하나의 프로세스만 사용 가능 |
| **점유와 대기 (Hold and Wait)** | 자원을 점유한 상태에서 다른 자원을 기다림 |
| **비선점 (Non-Preemption)** | 다른 프로세스가 사용 중인 자원을 강제로 빼앗을 수 없음 |
| **환형 대기 (Circular Wait)** | 프로세스 간 자원 요구가 원형으로 연결됨 |

### 해결 기법

| 기법 | 설명 |
| :--- | :--- |
| **예방 (Prevention)** | 발생 조건 4가지 중 하나 이상을 부정하여 사전에 방지 |
| **회피 (Avoidance)** | 안전한 상태에서만 자원 할당. **은행원 알고리즘(Banker's Algorithm)** |
| **발견/탐지 (Detection)** | 교착상태 발생을 허용하고, 발생 시 탐지하여 조치. 자원 할당 그래프 사용 |
| **회복 (Recovery)** | 교착상태 발생 후 프로세스 종료 또는 자원 선점으로 복구 |

## 5. 메모리 관리

### 가상 메모리 기법

| 기법 | 설명 |
| :--- | :--- |
| **페이징 (Paging)** | 가상 메모리를 **고정 크기의 블록(페이지)**으로 나누어 관리. 내부 단편화 발생 가능 |
| **세그먼테이션 (Segmentation)** | 가상 메모리를 **가변 크기의 논리적 단위(세그먼트)**로 나누어 관리. 외부 단편화 발생 가능 |

> **내부 단편화:** 할당된 메모리 공간 내부에 사용되지 않는 남는 공간이 생기는 현상
> **외부 단편화:** 할당/해제가 반복되면서 메모리 사이에 사용할 수 없는 작은 빈 공간이 흩어지는 현상

### 스래싱, 워킹셋, 지역성

| 용어 | 설명 |
| :--- | :--- |
| **스래싱 (Thrashing)** | 페이지 부재(Page Fault)가 과도하게 발생하여 프로세스 실행보다 **페이지 교체에 더 많은 시간**을 소비하는 현상. 다중 프로그래밍 정도가 높아질수록 발생 가능 |
| **워킹셋 (Working Set)** | 일정 시간 동안 프로세스가 자주 참조하는 **페이지들의 집합**. 워킹셋을 메모리에 유지하면 스래싱 방지 가능 |
| **지역성 (Locality)** | 프로세스가 기억장치를 참조할 때 일부 영역에 집중적으로 접근하는 경향 |

### 지역성의 종류

| 종류 | 설명 |
| :--- | :--- |
| **시간 지역성 (Temporal Locality)** | 최근 사용된 데이터가 **가까운 시간 내에 다시 사용**될 가능성이 높음. 예: 반복문, 스택 |
| **공간 지역성 (Spatial Locality)** | 사용된 데이터의 **인접한 주소의 데이터**가 사용될 가능성이 높음. 예: 배열 순차 접근 |

### 페이지 교체 알고리즘
페이지 부재(Page Fault) 발생 시 메모리에서 교체할 페이지를 선택하는 알고리즘입니다.

| 알고리즘 | 설명 |
| :--- | :--- |
| **OPT (Optimal)** | 앞으로 **가장 오랫동안 사용되지 않을** 페이지를 교체. 이론적 최적, 실현 불가 |
| **FIFO (First In First Out)** | 메모리에 **가장 먼저 들어온** 페이지를 교체. 벨레이디의 모순 발생 가능 |
| **LRU (Least Recently Used)** | **가장 오래 전에 사용된** 페이지를 교체. 가장 많이 사용되는 알고리즘 |
| **LFU (Least Frequently Used)** | **사용 빈도가 가장 낮은** 페이지를 교체 |
| **NUR (Not Used Recently)** | 최근에 사용하지 않은 페이지를 교체. 참조 비트와 변형 비트 사용 |

> **벨레이디의 모순 (Belady's Anomaly):** FIFO에서 프레임 수를 늘렸는데 오히려 페이지 부재가 증가하는 현상

## 6. 디스크 스케줄링

| 알고리즘 | 설명 |
| :--- | :--- |
| **FCFS** | 요청 순서대로 처리 |
| **SSTF (Shortest Seek Time First)** | 현재 헤드 위치에서 가장 가까운 요청 먼저 처리. 기아 현상 가능 |
| **SCAN (엘리베이터)** | 한쪽 끝까지 이동하며 처리 후 반대 방향으로 이동 |
| **C-SCAN** | 한쪽 방향으로만 이동하며 처리. 끝에 도달하면 처음으로 되돌아감 |
| **LOOK** | SCAN과 유사하지만 요청이 있는 곳까지만 이동 |

## 7. 네트워크 OSI 7계층
*(암기: 물·데·네·전·세·표·응)*

| 계층 | 이름 | 주요 기능 | 대표 장비/프로토콜 | 데이터 단위 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **물리** | 비트 스트림 전기적 신호 전송 | 리피터, 허브 | 비트(Bit) |
| **2** | **데이터 링크** | 노드 간 신뢰성 있는 전송, MAC 주소 | 브릿지, 스위치 | 프레임(Frame) |
| **3** | **네트워크** | 최적 경로 설정(Routing), IP 주소 | 라우터 / IP, ICMP, ARP | 패킷(Packet) |
| **4** | **전송** | 종단 간(End-to-End) 신뢰성 보장, 흐름·오류 제어 | 게이트웨이 / TCP, UDP | 세그먼트(Segment) |
| **5** | **세션** | 연결 유지 및 동기화 | SSH, TLS | 데이터 |
| **6** | **표현** | 압축, 암호화, 포맷 변환 | JPEG, MPEG | 데이터 |
| **7** | **응용** | 사용자에게 네트워크 서비스 제공 | HTTP, FTP, SMTP, DNS | 데이터 |

### HDLC 프레임과 모드 ★2026년 1회차★

HDLC(High-level Data Link Control)는 데이터 링크 계층에서 사용하는 비트 중심 전송 제어 프로토콜이다.

| 구분 | 설명 | 시험 포인트 |
| :--- | :--- | :--- |
| **정보 프레임(I-frame)** | 사용자 데이터 전송 | 실제 데이터 전달 |
| **감독 프레임(S-frame)** | 흐름 제어, 오류 제어, 수신 확인 | 제어/확인용 |
| **비번호 프레임(U-frame)** | 링크 설정, 해제, 모드 설정 | 링크 관리용 |
| **비동기 균형 모드(ABM)** | 두 장비가 대등한 관계로 명령과 응답을 모두 수행 | 균형 = 대등 |
| **비동기 응답 모드(ARM)** | 종국이 주국의 명령 없이도 응답 전송 가능 | 응답 = 종국 응답 가능 |

## 8. IP 주소 체계

### IPv4 vs IPv6

| 구분 | IPv4 | IPv6 |
| :--- | :--- | :--- |
| **주소 길이** | **32비트** | **128비트** |
| **주소 표기** | 10진수 (192.168.0.1) | 16진수 (2001:0db8::) |
| **전송 방식** | 유니캐스트, 멀티캐스트, **브로드캐스트** | 유니캐스트, 멀티캐스트, **애니캐스트** |
| **보안** | IPSec 선택 | IPSec **기본 내장** |
| **헤더 크기** | 가변 | 고정 (40바이트) |

### IP 클래스

| 클래스 | 범위 | 네트워크/호스트 비트 | 용도 |
| :--- | :--- | :--- | :--- |
| **A** | 0.0.0.0 ~ 127.255.255.255 | 8 / 24 | 대규모 네트워크 |
| **B** | 128.0.0.0 ~ 191.255.255.255 | 16 / 16 | 중규모 네트워크 |
| **C** | 192.0.0.0 ~ 223.255.255.255 | 24 / 8 | 소규모 네트워크 |

### 서브넷 마스크
* IP 주소에서 **네트워크 부분과 호스트 부분**을 구분하는 32비트 값
* 서브넷 마스크가 255.255.255.0 (/24)이면: 호스트 비트 = 8비트, 사용 가능한 호스트 수 = 2^8 - 2 = **254개** (네트워크 주소, 브로드캐스트 주소 제외)

### /23 서브넷 네트워크 주소 계산 ★2026년 1회차★

`255.255.254.0`은 `/23`이다. 세 번째 옥텟의 블록 크기는 `256 - 254 = 2`이므로 네트워크 범위가 2씩 증가한다.

| IP 주소 | 세 번째 옥텟 범위 | 네트워크 주소 |
| :--- | :--- | :--- |
| `192.168.11.20/23` | `10 ~ 11` | `192.168.10.0` |
| `192.168.12.200/23` | `12 ~ 13` | `192.168.12.0` |

* `/23` 범위 예: `192.168.10.0 ~ 192.168.11.255`
* 네트워크 주소는 해당 범위의 첫 주소이고, 브로드캐스트 주소는 해당 범위의 마지막 주소이다.

### TCP vs UDP

| 구분 | TCP | UDP |
| :--- | :--- | :--- |
| **연결 방식** | 연결 지향형 (3-Way Handshake) | 비연결형 |
| **신뢰성** | 신뢰성 보장 (재전송, 순서 제어) | 신뢰성 보장 안 함 |
| **속도** | 상대적으로 느림 | 빠름 |
| **용도** | 파일 전송, 웹 브라우징, 이메일 | 실시간 스트리밍, 온라인 게임, DNS |

### 3-Way Handshake (TCP 연결 수립)
1. 클라이언트 → 서버: **SYN** (연결 요청)
2. 서버 → 클라이언트: **SYN + ACK** (요청 수락)
3. 클라이언트 → 서버: **ACK** (연결 확인)

| **top** | 실시간 프로세스 상태 모니터링 |

### 권한 관련

| 명령어 | 설명 |
| :--- | :--- |
| **chmod** | 파일/디렉터리의 **접근 권한** 변경 |
| **chown** | 파일/디렉터리의 **소유자** 변경 |
| **chgrp** | 파일/디렉터리의 **소유 그룹** 변경 |

### 파일 권한 (rwx)
```
rwx rwx rwx  =  소유자 / 그룹 / 기타 사용자
r(읽기)=4, w(쓰기)=2, x(실행)=1
```
* 예: `chmod 755 파일명` → 소유자(rwx=7), 그룹(r-x=5), 기타(r-x=5)
* 예: `chmod 644 파일명` → 소유자(rw-=6), 그룹(r--=4), 기타(r--=4)

## ★ 핵심 암기 포인트 ★

1. **OS 목적:** 처리 능력↑, 반환 시간↓, 사용 가능도↑, 신뢰도↑
2. **비선점:** FCFS, SJF, HRN, 우선순위 / **선점:** RR, SRT, 다단계 큐, 다단계 피드백 큐
3. **HRN 공식:** (대기시간 + 서비스시간) / 서비스시간
4. **교착상태 4조건:** 상호배제, 점유와 대기, 비선점, 환형 대기
5. **교착상태 해결:** 예방(부정), 회피(은행원), 발견(탐지), 회복
6. **페이지 교체:** OPT(최적/불가), FIFO(벨레이디 모순), LRU(가장 오래 전 사용), LFU(최소 빈도)
7. **OSI 7계층:** 물(비트)-데(프레임)-네(패킷)-전(세그먼트)-세-표-응
8. **HDLC 프레임:** 정보(I)=데이터, 감독(S)=흐름/오류/수신확인, 비번호(U)=링크 설정/해제/모드
9. **IPv4=32비트/브로드캐스트, IPv6=128비트/애니캐스트/IPSec 내장**
10. **/23=255.255.254.0:** 세 번째 옥텟이 2씩 증가, `192.168.11.x`의 네트워크 주소는 `192.168.10.0`
11. **TCP=연결지향/신뢰성, UDP=비연결/빠름**
12. **RIP=벨만-포드/홉15, OSPF=다익스트라/대규모, BGP=AS간**
13. **chmod:** r=4, w=2, x=1 (755=rwxr-xr-x)
14. **주요 포트:** FTP(20/21), SSH(22), HTTP(80), HTTPS(443), DNS(53), SMTP(25)
15. **스래싱:** 페이지 부재 과다 → 페이지 교체에 시간 소비 → 워킹셋으로 방지
16. **지역성:** 시간 지역성(최근 사용 데이터 재사용), 공간 지역성(인접 데이터 사용)
17. **페이징=고정크기/내부단편화, 세그먼테이션=가변크기/외부단편화**
